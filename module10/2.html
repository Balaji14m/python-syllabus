Classes & Objects

Class  


Class is a blueprint for an object sometimes it is also known as class is a prototype for an object.

When the thought of managing objects came into mind we went with class to define the structure of it. 

Difference between class vs object 

A class is a blueprint or prototype that defines the variables and the methods (functions) common to all objects of a certain kind.

An object is a specimen of a class. Software objects are often used to model real-world objects you find in everyday life.

Steps to make a Class:  

  • Classes are created by keyword class.

  • Attributes are the variables that belong to a class.

  • Attributes are always public and can be accessed using the dot (.) operator.

Object 

Object is a real world entity whatever we see around us. Object is also a user defined data type.

An object consists of : 

  • State: It is represented by the attributes of an object. It also reflects the properties of an object.

  • Behavior: It is represented by the methods of an object. It also reflects the response of an object to other objects.

  • Identity: It gives a unique name to an object and enables one object to interact with other objects.
