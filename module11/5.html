Custom Exception

Python has many built-in exceptions which force your program to output a mistake when something in it goes wrong.

However, sometimes you'll get to create custom exceptions that serve your purpose.

In Python, users can define such exceptions by creating a replacement class. This exception class has got to be derived, either directly or indirectly, from Exception class. Most of the built-in exceptions also are derived from this class.

>>>classCustomError(Exception):

...pass

...

 

>>>raiseCustomError

Traceback(most recent call last):

...

__main__.CustomError

 

>>>raiseCustomError("An error occurred")

Traceback(most recent call last):

...

__main__.CustomError:An error occurred



Here, we've created a user-defined exception called CustomError which springs from the Exception class. These new exceptions are often raised, like other exceptions, using the raise statement with an optional error message.

When we are developing an outsized Python program, it's an honest practice to put all the user-defined exceptions that our program raises during a separate file. Many standard modules do that By defining exceptions separately as exceptions.py or errors.py (generally but not always).

Let�s now develop a User-defined exception class. It can implement everything a traditional class can do, but we generally make them simple and concise. Most applications are declared as a custom base class and derive other exception classes from this base class. This idea is formed clearer within the following example.

Example: User-Defined Exception in Python

In this example, we'll illustrate how user-defined exceptions are often utilized in a program to boost and catch errors.

This program will ask the user to enter variety until they guess a stored number correctly. to assist them to figure it out, a hint is provided whether their guess is bigger than or but the stored number.

class Error(Exception):

    """Base class for other exceptions"""

    pass

class ValueTooSmallError(Error):

    """Raised when the input value is too small"""

    pass

class ValueTooLargeError(Error):


    """Raised when the input value is too large"""

    pass


 # our main program

# user guesses a number until he/she gets it right

# you need to guess this number

number=10

while True:

    try:

        i_num=int(input("Enter a number: "))

        if i_num < number:

            raise ValueTooSmallError;

        elif i_num> number:

            raise ValueTooLargeError;

            break

        break

    except ValueTooSmallError:

        print("This value is too small, try again!")

        print()

    except ValueTooLargeError:

        print("This value is too large, try again!")

        print()

print("Congratulations! You guessed it correctly.")


Here, we have defined a base class called Error.

A critical operation that can raise an exception is placed inside the try clause and the code that handles exceptions is written in the except clause.

It is up to us, what operations we perform once we have caught the exception.

In this program, we loop until the user enters an integer that has a valid reciprocal. The portion that can cause an exception is placed inside the try block.

If no exception occurs, except block is skipped and normal flow continues. But if any exception occurs, it is caught by the except block.

Here, we print the name of the exception using ex_info() function inside sys module and ask the user to try again. We can see that the values 'a' and '1.3' causes ValueError and '0' causes ZeroDivisionError.
